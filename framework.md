## 模块设计说明

### 1. main.cpp

程序入口，仅负责系统初始化与启动菜单。

职责：

-   加载数据文件
-   创建图对象
-   启动菜单循环

------------------------------------------------------------------------

### 2. City 模块

表示城市信息的数据模型。

包含字段：

-   id
-   name
-   x, y 坐标
-   description

特点：

-   纯数据类
-   不包含算法逻辑

------------------------------------------------------------------------

### 3. Edge 模块

表示通信线路。

包含：

-   起点编号
-   终点编号
-   权值（通信距离）

通信距离通过欧几里得距离计算并四舍五入。

------------------------------------------------------------------------

### 4. Graph 模块（核心数据结构）

负责维护通信网络图。

主要功能：

-   保存城市集合
-   维护邻接表
-   添加/删除城市与边
-   查询节点信息

Graph 仅负责数据存储，不实现复杂算法。

------------------------------------------------------------------------

### 5. GraphAlgorithms 模块

集中实现所有图算法。

包含算法：

-   连通性判断（BFS / 并查集）
-   最短路径（Dijkstra）
-   最小新增线路（Kruskal）
-   TSP近似算法
-   Steiner树近似算法

设计思想：

算法与数据结构分离。

------------------------------------------------------------------------

### 6. FileManager 模块

负责系统持久化。

功能：

-   从文件读取城市与线路
-   保存修改后的图数据
-   数据格式解析

------------------------------------------------------------------------

### 7. Menu 模块

实现用户交互界面。

职责：

-   显示菜单
-   接收用户输入
-   调用对应功能

属于控制层（Controller）。

------------------------------------------------------------------------

### 8. Utils 模块

工具函数集合，例如：

-   距离计算
-   输入校验
-   字符串处理
-   数学辅助函数

------------------------------------------------------------------------

## 模块依赖关系

正确依赖方向：

    Menu
      ↓
    GraphAlgorithms
      ↓
    Graph
      ↓
    City / Edge

FileManager 可访问 Graph，但不依赖算法模块。

------------------------------------------------------------------------

## 程序运行流程

    main()
       ↓
    读取文件
       ↓
    建立Graph
       ↓
    进入菜单系统
       ↓
    执行算法操作
       ↓
    保存数据

------------------------------------------------------------------------

## 核心算法映射

  功能           算法
  -------------- ----------------
  连通性判断     并查集
  最小新增线路   Kruskal
  单源最短路径   Dijkstra
  遍历全部城市   TSP近似
  空白布线方案   MST近似Steiner
