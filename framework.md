# 通信网络设计系统项目框架

> 【题目1】通信网络设计问题
> 将某国的国土简化为欧几里得二维平面，某国有N个城市。我们将一个城市看作一个顶点。两> 个顶点之间的边是两个城市之间的通信线路。我们考虑简单图。也就是说，每两个顶点之间最> 多只有一条边。
> 基本功能及要求如下：
1、提供用户操作的菜单和界面，实现系统的所有功能。不要求，但是鼓励实现图形界面。
2、添加城市的信息，具体信息包括：编号、名称、横纵坐标（均为整数）、简介等。
3、添加通信线路信息，具体信息包括：起点编号、终点编号等。将通信线路简化为直线。通信线路的长度采用勾股定理进行计算，四舍五入取整。
4、以上添加的城市和通信线路信息可以修改和删除，即具备信息的增删改功能。并且所有信息要求以文件的形式存储（例如文本文件），格式自行设计。当下次运行程序时，从文件读取城市和通信线路信息。
5、判断该国的所有城市是否构成连通图。采用深度优先搜索、广度优先搜索、并查集均可。如果该国的所有城市不构成连通图，那么在保留现有通信线路的前提下，增加哪些线路使得该国的所有城市构成连通图？目标是增加的线路长度和最小。请你设计并实现算法。
6、给定一个城市，查询该城市到其他任一城市的最短简单通信路径及距离，并按照距离从近到远的顺序显示。
7、求从某一城市出发经过该国所有城市（城市可以重复）且距离最短的路线；求从某一城市出发经过该国所有城市（城市可以重复）并回到源点的最短路线。
8、给定N个城市的坐标，假设我们完全从空白开始，也就是说，没有已建好的线路，请你设计一个最短的通信线路布线方案，使得N个城市连通。我们将每个城市看作一个顶点，除了已知的N个顶点以外，我们还可以添加辅助的顶点。将每两个顶点之间的线路简化为直线。
提示：第7问、第8问是NP难度问题。第7问是旅行商问题（Traveling Salesman Problem）的变形。第8问是施泰纳最小树问题（Steiner Shortest Tree Problem）。建议同学们在独立思考的基础上查阅文献（但不能完全依赖文献）。

## 模块设计说明

### 1. main.cpp

程序入口，仅负责系统初始化与启动菜单。

职责：

-   加载数据文件
-   创建图对象
-   启动菜单循环

------------------------------------------------------------------------

### 2. City 模块

表示城市信息的数据模型。

包含字段：

-   id
-   name
-   x, y 坐标
-   description

特点：

-   纯数据类
-   不包含算法逻辑

------------------------------------------------------------------------

### 3. Edge 模块

表示通信线路。

包含：

-   起点编号
-   终点编号
-   权值（通信距离）

通信距离通过欧几里得距离计算并四舍五入。

------------------------------------------------------------------------

### 4. Graph 模块（核心数据结构）

负责维护通信网络图。

主要功能：

-   保存城市集合
-   维护邻接表
-   添加/删除城市与边
-   查询节点信息

Graph 仅负责数据存储，不实现复杂算法。

------------------------------------------------------------------------

### 5. GraphAlgorithms 模块

集中实现所有图算法。

包含算法：

-   连通性判断（BFS / 并查集）
-   最短路径（Dijkstra）
-   最小新增线路（Kruskal）
-   TSP近似算法
-   Steiner树近似算法

设计思想：

算法与数据结构分离。

------------------------------------------------------------------------

### 6. FileManager 模块

负责系统持久化。

功能：

-   从文件读取城市与线路
-   保存修改后的图数据
-   数据格式解析

------------------------------------------------------------------------

### 7. Menu 模块

实现用户交互界面。

职责：

-   显示菜单
-   接收用户输入
-   调用对应功能

属于控制层（Controller）。

------------------------------------------------------------------------

### 8. Utils 模块

工具函数集合，包含不属于某一特定对象的通用辅助逻辑。

当前实现示例：

-   距离计算：`calDistance` 使用城市坐标计算欧几里得距离并四舍五入。
-   字符串处理：`split` 按指定分隔符拆分字符串。
-   输入校验：`isInteger` 判断文本是否为有效整数。
-   未来可扩展数学函数、格式转换、校验等。

------------------------------------------------------------------------

## 模块依赖关系

正确依赖方向：

    Menu
      ↓
    GraphAlgorithms
      ↓
    Graph
      ↓
    City / Edge

FileManager 可访问 Graph，但不依赖算法模块。
